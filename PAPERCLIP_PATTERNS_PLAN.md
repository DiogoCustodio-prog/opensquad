# Plano — Foundation de padrões Paperclip no Opensquad

## Objetivo
Incorporar no Opensquad os padrões de operação do Paperclip sem trocar a arquitetura principal do framework.

## Escopo desta fase (Fase 1)
1. Definir modelo de tickets por execução de squad.
2. Definir herança de objetivo (goal ancestry) para cada run.
3. Definir guardrails de orçamento por execução/agent.
4. Definir governança (aprovar, pausar, retomar, encerrar).
5. Definir trilha de auditoria mínima por execução.

## Contratos mínimos
- Ticket
- Goal ancestry
- Budget guardrail
- Governança
- Auditoria

## Critérios de aceite
- Especificação clara dos contratos
- Sequência incremental sem quebrar fluxo atual
- Checklist de validação para cada padrão
